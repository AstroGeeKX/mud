好的，我们针对你的需求（**不使用LVM**且最省事地将空闲的10GB扩容到`/`目录所在的`/dev/sda3`）分步解释：

---

### 关键前提条件
1. **空闲空间必须与`/dev/sda3`相邻**（物理磁盘地址连续，且在`sda3`之后）。
2. **当前系统支持在线扩容**（部分文件系统如ext4/xfs支持在线调整，但需内核支持）。
3. **操作前备份数据**（任何分区操作都有风险）。

---

### 操作步骤（以ext4文件系统为例）

#### 1. 确认分区布局
查看当前分区结构：
```bash
sudo fdisk -l /dev/sda
```
输出示例：
```
Device     Boot   Start       End   Sectors  Size Id Type
/dev/sda1  *       2048   1050623   1048576  512M 83 Linux
/dev/sda2       1050624  52430847  51380224 24.5G 82 Linux swap
/dev/sda3      52430848 104857599  52426752   25G 83 Linux  # 当前根分区
```
假设你有未分配的**10GB空闲空间**紧跟在`sda3`之后（即`/dev/sda3`的结束位置到磁盘末尾之间有10GB未使用）。

---

#### 2. 调整分区大小（无需格式化）
使用 `parted` 工具直接调整分区边界：
```bash
sudo parted /dev/sda
```
在 `parted` 交互界面：
```plaintext
(parted) resizepart 3    # 调整分区3（即sda3）的结束位置
End? [旧结束位置] -> 输入新的结束位置（如原25G+10G=35G，则输入35GB对应的扇区值）
(parted) quit
```

---

#### 3. 扩展文件系统
调整分区后，文件系统仍停留在原大小，需扩展：
```bash
sudo resize2fs /dev/sda3   # 针对ext4文件系统
```
如果是xfs文件系统：
```bash
sudo xfs_growfs /          # 直接对挂载点操作（需在线执行）
```

---

#### 4. 验证结果
```bash
df -h /
```
输出应显示根分区已扩容至35GB。

---

### 注意事项
1. **空闲空间必须紧邻**：若空闲空间不连续，需先通过 `parted` 或 `gdisk` 移动分区（耗时且高风险）。
2. **Live CD备用方案**：如果根分区无法在线调整，需通过Live USB启动系统后操作。
3. **文件系统兼容性**：ext2/3/4和xfs支持在线扩容，其他文件系统可能需卸载分区操作。
