我想测试一个并发的程序，服务器迭代，客户端也迭代，反复连接，算了，描述起来有点困难。已经写好客户端了，循环迭代连接服务器echo后断开连接，进行下一次循环。可以测得到的是1秒钟4400条消息，但是同时打开多个客户端，时间不变，说明这不是服务器的极限，而是客户端的问题。搞清楚这一点并不容易，尤其是在学习初期，这得多花时间反复磨练，而不是看到网上说什么并发量吞吐量，就以为是很简单的概念。比如我可以抛出很多不同的概念定义，同时连接数量叫并发量，那么反复连接echo再断开，这叫什么性能指标呢。并发量和数据处理量又有什么关系吗，根据数据量的不同，处理的时间也会不一样。

int a = 0;    const在*的左边/右边    左定值，右定向
int &ref = a;  引用等效定向指针  int *const p = &a; // p指针永远指向a
所以可以看作引用符号 后面永远有一个看不见的const，即 int &const ref = a;
